Notebook 3 - Getting the initial parameters values

This notebook is used to either indicate the initial parameter values (before calibrating them) or find them for a given species using an combination of available datasets and the advice of Eric Gustafson in the user guide of PnET-Succession v5.1.

Finding these initial values for your tree species might be the most work you will have to do here, especially for the “Core species parameters” (see below). Since I’ve made these notebooks with the goal of calibrating as many canadian tree species as possible, you’ll find sources of data and functions to compute some parameters for canadian tree species. But if you’re in the US or in Europe, or in another part of the world where previous LANDIS-II studies are scarce, you will have to to some digging.

**At the end of this notebook, you should have three .json files containing your parameters for each species, in the format used by the rest of the functions in these notebooks : one for the core species parameters, one for the PnET species parameters, and one for the PnET generic parameters (see below for explanations about the terms). These three .json files should be in the folder SpeciesParametersSets/Initial/.

Tip

Remember that as much as possible, it is best to get parameters values for your species in your study area, or near your study area, to avoid confusion with local variants or phenotypes. See Notebook 0.1 - Some important notions before we start for more.

Of course, some parameters (especially some core species parameters which represent global life history traits) don’t need to be local; and for other, you will have a lot of difficulty finding more local estimates. It’s alright, and it’s part of the complexity in calibrating a complex model such as LANDIS-II and PnET-Succession. Just take the best values you find, note the limitations and potential implications or your choices on what the model will produce, and carry on !

What parameters are we talking about here, and what are the different categories of parameters ?¶

In this page, we will talk about :

  • Core species parameters : The species parameters used by the LANDIS-II core (e.g. Longevity, dispersal distance, etc.). These will go in the “core” species parameter file for LANDIS-II (indicated in the main scenario file).

  • PnET species parameters : The species parameters used by PnET Succession that will vary between species. These will go into the species parameters file of PnET Succession.

  • PnET generic species parameters : The species parameters used by PnET Succession that we will keep constant between species. These will go into the generic parameter file of PnET Succession. See below for recommendations on which parameters to put in this category.

  • PnET generic parameters : Other generic parameters of PnET Succession that are not related to species, but that are still crucial for the functionning of PnET Succession (PARunits, PrecipEvents, etc.).

Why should some species parameters be “generic” and constant accross species in PnET Succession ?

Gustafson and Miranda (2023) advises that several species parameters should be used as generic species parameters, meaning that their value should be the same between species, in order to facilitate the comprehension of competition processes. Quoting from them :

Use the GenericPnETSpeciesParameters file for all parameters that are not varied among species. Set parameters that will be held constant for your particular experiment or study (e.g., MaintResp, InitialNSC, etc.)

Parameters that the model is highly sensitive to (for which common values should be used as much as possible) include: MaintResp, AmaxA, AmaxB, k, and FracBelowG.

Minimizing species differences in parameters such as SLWDel, PsnAgeRed, k, MaintResp, DNSC and FracBelowG, will make competitive interactions more predictable. Hold these as close to each other as possible, varying other parameters to calibrate as much as possible within empirical limits.

Which PnET species parameters should be generic ?

The choice depends on your species. As recommanded above by Gustafson and Miranda (2023), using as many parameters as you can as constant and generic accross species will make competition and other processes in PnET Succession easier to interpret and understand. But sometimes, you will have to change these parameters between species so as to properly represent their differences.

Here, I will start with the recommandations of Gustafson and Miranda (2023) and make MaintResp, InitialNSC, k, FracBelowG, PsnAgeRed, DNSC similar between species. SLWDel, k, AmaxA and AmaxB will not be generic as they have to vary at least between evergreen and deciduous according to Gustafson and Miranda (2023).

The rest of the species parameters (see here for a full list) like FolN, SLWmax, TOFol, TORoot, TOWood, HalfSat, H1, H2, H3, H4, LeafOnMinT, PsnTMin, PsnTOpt, EstMoist, EstRad, FracFol, FrActWd, SLWDel, k, AmaxA and AmaxB will change by species or by growth form/strategy (e.g. shade tolerant, etc.).

What parameters are we not using, currently ?¶

Currently, I’m avoiding most optional parameter to reduce the complexity of the calibration. Here are the parameters that we will avoid.

  • MaxLAI: Maximum leaf area index (LAI) that a species can achieve in its lifetime (for an age cohort ?).

  • MaxFolN: Controls the dynamic response of foliar nitrogen (FolN) to light.

  • FolNShape: Controls the dynamic response of foliar nitrogen (FolN) to light.

  • PsnTMax: Maximum average daytime temperature for photosynthesis.

  • ColTol: Cold tolerance (coldest temperature that the species can survive).

  • CO2HalfSatEff: Slope coefficient used to reduce HalfSat (= increase shade tolerance) as CO2 concentration increase.

  • CO2AMaxBEff: Proportional change in AmaxB at 550ppm CO2 relative to 350ppm.

  • O3StomataSens: Species susceptibility of the stomata to ozone-induced sluggishness.

  • O3GrowthSens: Species susceptibility to ozone-induced tissue damage.

  • LifeForm: Specifies the life form (stature) of the species to determine the limits to the species major canopy layer (needed if you want to simulate shrubs, which is a possibility !)

  • MossScalar: Unit converter from biomass to depth in the case of a pseudo species (moss).

However, we’re most likely going to use these two optional parameters to better distinguish between pioneer and later successional species :

  • MaxFracFol: Controls the dynamic response of the fraction of Foliage (FracFol) to light.

  • FracFolShape: Controls the dynamic response of the fraction of Foliage (FracFol) to light.

You can, of course, choose to set specific values for these parameters depending on your research question.

Change in core species parameters between LANDIS-II v7 and LANDIS-II v8

The v7 of LANDIS-II has been used by a lot of studies in North America; but with the recent (2025-2026) switch to the version 8 of the model, the core species parameters have been changed : shade tolerance and fire tolerance have been removed, the vegetative reproduction is no longer a “yes/no” but a probability and you must indicate the sprouting ages, etc.

So if you take a look at previous studies with LANDIS-II, you will most likely find the old format of parameters, and you might need to do a bit more research to find the new ones.

What do I do if I can’t find published parameters for a given tree species ? Where do I find initial parameter values ?

This is the though part. The big problem with the PnET models (as explanation in the user guide) is that they are described as models easy to parameterize because you can just use empirical values as parameters; but strangely, it’s extremely hard to find empirical values for the parameters used in the model, even just the most important ones !

So you might find yourself without any good source of parameters for a species you want to calibrate. If you find yourself in this kind of situation, there is a section at the end of this notebook that will allow you to generate the parameters for a tree species based on its shade tolerance score (from 1 to 5) and its evergreen/deciduous type. This is done using the advice of Eric Gustafson in the PnET-Succession user guide; it’s far from perfect, but it should produce a parameter set that makes sense.

However, you will still need estimates of the growth curve in ideal conditions (see next notebook) and longevity data to be able to calibrate your species. And if you can find parameters or empirical values for the parameters in the litterature, you should try to do so.

Format of the following tables of parameters

As the end of this notebook is a code section that will read the following tables of parameters and transform them in Python dictionnaries that can be used by the functions of functionsForCalibration.py to create LANDIS-II parameter files, which will be used to automatize LANDIS-II simulations further on.

So that the function can read the tables properly, the content of each cell must respect the following format :

  • The cell must start with the value of the parameter

  • All other content must be either in footnote format (between square brackets []), between parenthesis, or after an asterisk. This content will then be ignored by the function to just keep the value at the beginning of the cell.

Example :

  • 0.35 (average among species in [^2] and [^3]) : Good. Only the value 0.35 will be taken; the rest, between parenthesis, will be ignored.

  • 0.35 : average among species in [^2] and [^3] : Not good. The rest of the cell appart from the value (0.35) should be between parenthesis or square brackets.

About the LayerThreshRatio parameter

LayerThreshRatio is a generic parameter that is used to define how easily age cohorts of the same species are grouped in the same canopy layer.

I’m still having trouble wrapping my mind about how the parameter works exactly; what’s certain is that it has a large effect on the outcomes of simulations. Increasing it seems to let more cohorts cohabitate together in a cell, especially young and old cohorts. Might be because in this case, young cohorts tend to be grouped with older cohorts more easily, and they thus receive more light according to the algorithm.

It’s difficult for me to recommend the right value for this parameter. For now, I will put it at 0.67, as it’s one of the values recommanded by Eric Gustafson in the user guide, and because it seems to allow more room for complex dynamics inside the cells rather than a periodic domination by large, old cohorts. But you might want to change it if you’re seeing strange results of cohabitations between cohort in a given cell.

Getting drought and waterlogging tolerance parameter values from the litterature using regression¶

In PnET-Succession, 4 parameters are related to the effect of water in the soils on tree growth : H1, H2, H3 and H4.

These parameters together define a curve relating the pressure head in the soil with the value of fWater, which is the photosynthesis reduction factor linked to water. Here is an example from the user guide :

The idea of the trapezoid is that the photosynthesis of tree is overall very little affect by the amount of water present in the soil (fWater = 1, no effect on the photosynthesis); unless we arrive at thresholds of waterlogging or drought (pressure head too small or too high) where the photosynthesis will abruptly diminish. The user guide even recommends to use a value of H3 and H4 (the parameters related to high values of pressure head = drought conditions) to be the same, so that photosynthesis simply stops in drought months.

Warning

Be careful about the signs (positive or negative) used for the pressure head ! I’ve had a lot of confusion about this.

The pressure head is a measure of water pressure in the soil. It represents the following idea : If I put a cylinder in contact with the soil, will it be filled with water because there is so much water in the soil that it wants to get out (pressure, which happens during waterlogging); or will the soil instead “suck” water because it’s lacking water (suction, which happens during droughts) ?

The problem is that normally, a suction normally results in negative values of pressure head (if I had water in my cylinder, the water would go down in the soils); while pressure results in positive values (if the water pushes up into the cylinder because of pressure, how much does it pushes up ?). Pressure head is therefore the measurement, in units of distance, of how up or down the water goes when put in contact with the soil.

But in PnET-Succession, the sign convention is reversed by using the “absolute” values of pressure head. What that means is that in the case of PnET-Succession, the higher the pressure head, the more we are in drought; and vice versa. As such, the drought parameters H3 and H4 often have positive values above 100; while the waterlogging parameters H1 and H3 are often near 0, or even negative. Keep that in mind !

We sadly lack empirical data to get good estimates of these 4 parameters, and even more for many species. Although Gustafson et al. 2025 tried to use empirical/experimental data to derive values of H1 and H2 (waterlogging).

As such, ideally, we would calibrate this parameters to ensure that the tree species react in the right way on the right soils in the landscape.

Sadly, I haven’t found a way to do this calibration properly, and I’m lacking time. As such, I went with a simpler option :

We can simply create a regression (linear or non-linear) of the score of Drought and Waterlogging tolerance acquired from Niinemets and Valladares 2006 and previously published parameters (i.e. those the user guide of PnET-Succession, including those of Gustafson et al. 2025 which have been derived from an empiical experiment. This is quick and easy, and will make sure that we use parameters similar to previous studies.

The default of this approach is that we don’t do any calibration, and just re-use what have been done before. It’s also deeply reliant on the hypothesis that these parameters are completely dependent on a measure as fuzzy as the “drought tolerance” or “waterlogging tolerance” of a give tree species. It also will not produce parameters that are in any way contextual to our landscape (soils, precipitations, water in the soils); we just have to hope that the model will produce soil water content that make sense for our landscape (although this can be explored).

Still, I believe that this will do for now.

Getting temperature-related parameter values from the litterature using regression¶

These parameters are quite important as they regulate how the age cohorts react to change in temperature; PsnTMin, PsnTOpt and PsnTMax define a bell curve going from 0 (at PsnTMin and PsnTMax) to 1 (at PsnTOpt) to define fTemp, a reduction factor for the photosynthesis. This reduction factor will be multiplied by the theoretical photosynthesis done at a given month to reduce it. LeafOnMinT will define the minimum monthly temperature for leaves to be on the tree.

Because of their definition (i.e. PsnTMin being “The average monthly temperature under which no photosynthesis happen, etc.”), it’s quite hard to get values for these parameters. See notebook about future improvements for more discussion about it.

As my time is limited, I went with a simple approach of statistical regression based on already published values, and a new dataset of species temperature ranges published by Morgane Dendoncker from theme 1 in DIVERSE for many species in north america. See Dendoncker et al. 2026.

By exploring the data, I noticed some strong correlations between some variables of Dendoncker et al. 2026 and previously published values for these parameters. The causes of these correlation is quite obvious; for example, the fact that there is an important positive correlation between the 5th percentile of the average annual temperatures of where a tree species is located (one of the variables from Dendoncker et al. 2026) and PsnTMin is explained by the fact that at the northern edge of the species distribution (i.e. the 5th percentile of the temperatures in its range), the temperature becomes so low that the number of Growing Degrees Days (GDD) becomes close to what the species can survive, which implies that the average annual temperature becomes close to the minimum temperature for growth for the species.

I constructed 4 different GAMs that were the most performant at predicting PsnTMin/TOpt/TMax and LeafOnMinT based on variables from Dendoncker et al. 2026. The R squared are almost always above 0.9, indicating a very good fit, which we can see by comparing the values we predict compared to the ones used for training the model (in this case, I used the values published by Eric Gustafson in the user guide of PnET-Succession).

This approach has the benefits of allowing us to get these parameters for many different tree species, as the data from Dendoncker et al. 2026 spans a great number of species in North America. It will also allow us to be certain that the species are “ranked” between each other in terms of their temperature tolerance, and is rooted in empirical data (thanks to the work of Dendoncker et al. 2026). However, most of the approach relies on the values used by the training (here, the parameters of Eric Gustafson in the user guide). From discussion with him, Gustafson seem to have used maps of isoterms in the distribution of species that were available on the website of the initial PnET models. I was unable to find them.

In the end, I expect this simple approach to be quite robust, and to produce parameters that will make for some quite realistic growing seasons; but a more localized calibration based on a new methodology is certainty possible. See notebook about future improvments for a discussion on this.

SLWmax is derived from the published data of Diaz et al. 2022

With a bit of trial and error, I’ve noticed that the SLWmax values predicted from the propositions of Eric Gustafson in the user guide of PnET-Succession v5.1 were not always very good, especially for some peculiar species like the Tamarack (Larix laricina).

Therefore, I’ve downloaded and used the data of Diaz et al. 2022 which contains values of LMA (Leaf Mass Area), which is the same thing as the Specific Leaf Weight (SLW). I’ve taken the dataset and removed all non-tree species from it, and removed columns that were not useful to my use here.

However, what we need is SLWmax, while the measure in the Diaz et al. 2022 dataset is most likely the average SLW of leaves for a species.

To derive SLWmax from the LMA of Diaz et al. 2022 (that I will now call SLWavgSLW_{avg}SLWavg​), I therefore used the following equation that is based on the parameter SLWDel and the IMAX generic parameter :

SLWmax=SLWavg1−(SLWDel×IMAX−12)SLWmax = \frac{SLW_{avg}}{1 - \left( SLWDel \times \frac{IMAX - 1}{2} \right)}SLWmax=1−(SLWDel×2IMAX−1​)SLWavg​​

This formula accounts for the fact that PnET-Succession distributes foliage into IMAX with even proportions, and reduces the specific leaf weight linearly from the top sublayer (SLWmax) to the bottom sublayer based on the SLWDel proportion. For example, if SLWDel=0.2 (the standard value for deciduous species), the SLW of each subsequent sublayer decreases by 20% of the SLWmax value. This creates an arithmetic progression:

  • Sublayer 1: SLWmax×1.0

  • Sublayer 2: SLWmax×(1−SLWDel)

  • Sublayer 3: SLWmax×(1−2×SLWDel)

  • ...and so on for IMAX layers.

Therefore, using this formula means that the average of SLW accross the layers will be equal to the value found in Diaz et al. 2022 for the species for the given SLWmax we will find.

Not that while the dataset contains a measure of nitrogen in the leaves (Nmass), I haven’t used it to derive the FolN parameter. This is because in the user guide of PnET-Succession v5.1, Eric Gustafson says that it is more important to make sure that FolN is ranked relative to the shade tolerance of the species simulated, rather than using empirical values. Therefore, I believe that using the FolN scale that changes with the shade tolerance of a species as Gustafson proposes is a better way of doing things here.

    # If the species is not found in Niinemets and Valladares, we ask for a manual input.
    if additionalSpeciesNamesDictionnary[speciesName] not in list(niinemetsValladaresData["Species"]):
        print("Species " + str(additionalSpeciesNamesDictionnary[speciesName]) + " is not found in the data of Niinemets and Valladares (2006). Please enter the information of the species manually :")
        shade_tolerance = -999
        while shade_tolerance < 1 or shade_tolerance > 5:
            shade_tolerance = float(input("Please input the shade tolerance of " + str(additionalSpeciesNamesDictionnary[speciesName]) + " (from 1 = very intolerant to 5 = very tolerant"))
        tree_type = "none"
        while tree_type != "evergreen" and tree_type != "deciduous":
            tree_type = input("Please input the tree type (evergreen or deciduous) of " + str(additionalSpeciesNamesDictionnary[speciesName]) + " :")
        wood_type = "none"
        while wood_type != "hardwood" and tree_type != "softwood":
            wood_type = input("Please input the wood type type (hardwood or softwood) of " + str(additionalSpeciesNamesDictionnary[speciesName]) + " :")
    # If the species is in Niinemets and Valladares, we get the values from the dataset.
    else:
        rowWithSpecies = niinemetsValladaresData[niinemetsValladaresData["Species"] == additionalSpeciesNamesDictionnary[speciesName]]
        # Some values in the dataset of Niinemets and Valladares are inferior to 1 or superior to 5;
        # not sure why. Not explained in the article. We deal with these situations by cropping the values to 1-5.
        shade_tolerance = min(5 , max(1, list(rowWithSpecies["Shade tolerance"])[0]))
        if list(rowWithSpecies["Evergreen"])[0] == "Y":
            tree_type = "evergreen"
        else:
            tree_type = "deciduous"
        if list(rowWithSpecies["Gymnosperm"])[0] == "Y":
            wood_type = "softwood"
        else:
            wood_type = "hardwood"

    # This function uses the rules indicated by Eric Gustafson in the user guide of PnET-Succession v5.1.
    # There is two parameters from which Gustafson has no rule to get them : TOFol and FrActWd.
    # For these, I used the averaged values for deciduous/evergreen and hardwood/softwoods in previous publications of PnET-Succession parameters
    # this gives the following :
    # - TOFol : 1 for deciduous trees, 0.3 for evergreens
    # - FrActWd : 0.000035 for hardwoods, 0.000029 for softwoods

    # We retrieve the parameter IMAX, which is needed for computing SLWmax in the following function
    IMAX = json.load(open('./SpeciesParametersSets/Initial/InitialGenericParameters.json'))["IMAX"]
    resultDict[speciesName].update(get_parameters_gustafson_rules(shade_tolerance,
                                                                  tree_type,
                                                                  wood_type,
                                                                 additionalSpeciesNamesDictionnary[speciesName],
                                                                 IMAX))


    ############################################################
    # Get the values of H1/H2/H3/H4
    # Getting drought and wl tolerance from NiinemetsAndValladares
    # Again, if the species is not in the niinemets and valladares dataset, we ask for the data
    if additionalSpeciesNamesDictionnary[speciesName] not in list(niinemetsValladaresData["Species"]):
        print("Species " + str(additionalSpeciesNamesDictionnary[speciesName]) + " is not found in the data of Niinemets and Valladares (2006). Please enter the information of the species manually :")
        drought_tolerance = -999
        while drought_tolerance < 1 or drought_tolerance > 5:
            drought_tolerance = float(input("Please input the drought tolerance of " + str(additionalSpeciesNamesDictionnary[speciesName]) + " (from 1 = very intolerant to 5 = very tolerant"))
        waterlogging_tolerance = -999
        while waterlogging_tolerance < 1 or waterlogging_tolerance > 5:
            waterlogging_tolerance = float(input("Please input the waterlogging tolerance of " + str(additionalSpeciesNamesDictionnary[speciesName]) + " (from 1 = very intolerant to 5 = very tolerant"))

    # If the species is in Niinemets and Valladares, we get the values from the dataset.
    else:
        rowWithSpecies = niinemetsValladaresData[niinemetsValladaresData["Species"] == additionalSpeciesNamesDictionnary[speciesName]]
        drought_tolerance = min(5 , max(1, list(rowWithSpecies["Drought tolerance"])[0]))
        waterlogging_tolerance = min(5 , max(1, list(rowWithSpecies["Waterlogging tolerance"])[0]))
    
    resultDict[speciesName].update(predict_H_parameters(
    drought_tolerance=drought_tolerance,
    waterlogging_tolerance=waterlogging_tolerance,
    plot=False,
    gam_summary=False
    ))
    
    # Getting the values for the temperature parameters
    # if the species is not in the Dendoncker et al. dataset, we ask for the data
    if additionalSpeciesNamesDictionnary[speciesName] not in list(dendonckerEtAlData["Species"]):
        print("Species " + str(additionalSpeciesNamesDictionnary[speciesName]) + " is not found in the data of Dendoncker et al. (2026). Please enter the information of the species manually :")
        MAT_q05 = -9999
        while MAT_q05 < -273:
            MAT_q05 = float(input("Please input MAT_q05 (5th percentile of the Mean Annual Temperature for the entire spatial distribution range of the species " + str(additionalSpeciesNamesDictionnary[speciesName]) + ", i.e. the lowest annual temperatures in places where it survives) :"))
        
        MAT_q95 = -9999
        while MAT_q95 < -273:
            MAT_q95 = float(input("Please input MAT_q95 (95th percentile of the Mean Annual Temperature for the entire spatial distribution range of the species " + str(additionalSpeciesNamesDictionnary[speciesName]) + ", i.e. the highest annual temperatures in places where it survives) :"))
        
        MTWM_q95 = -9999
        while MTWM_q95 < -273:
            MTWM_q95 = float(input("Please input MTWM_q95 (95th percentile of the Mean temperature of the Warmest Month for the entire spatial distribution range of the species " + str(additionalSpeciesNamesDictionnary[speciesName]) + ", i.e. the highest mean temperature of the warmest month in places where it survives) :"))
        
    # If the species is in the dataset, we get the values from the dataset.
    else:
        rowWithSpecies = dendonckerEtAlData[dendonckerEtAlData["Species"] == additionalSpeciesNamesDictionnary[speciesName]]
        MAT_q05 = list(rowWithSpecies["MAT.q05"])[0]
        MAT_q95 = list(rowWithSpecies["MAT.q95"])[0]
        MTWM_q95 = list(rowWithSpecies["MTWM.q95"])[0]
    
    resultDict[speciesName].update(predict_temperature_parameters_single_species(
                        species_code = speciesName,
                        species_latin_name = additionalSpeciesNamesDictionnary[speciesName],
                        MAT_q05 = MAT_q05,
                        MAT_q95 = MAT_q95,
                        MTWM_q95 = MTWM_q95,
                        return_df = False,
                        plotAndSummaries = False)[0][speciesName])

    
    print(resultDict[speciesName])
    print("\n")
