Loading the functions and packages needed for the code of these notebooks

The following code sections are used to load the different functions I wrote to be able to parse, edit, write parameter for small LANDIS-II PnET simulations; but also to read the outputs and plot them. These functions are essential to automatise the calibration process used in this repository, so be sure to load them anytime you want to re-use some code of these notebooks.

The functions are written in the file functionsForCalibration.py.

Tip

If you are editing these notebooks for your own purposes and are trying to make a new Python function that requires a python package which is missing in the docker image used to run these notebooks (see Clean_Docker_LANDIS-II_8_AllExtensions_PnETCalibration), add your package to Clean_Docker_LANDIS-II_8_AllExtensions_PnETCalibration\files_to_help_compilation\pip_requirements.txt and rebuild this docker image.

The pip_requirements.txt file uses “fixed” packages versions in order to ensure that the code in these notebooks can run in the long-term (and not break because of package updates). If you don’t know which version to specify for your package, simply install your package(s) first with pip in a terminal launched from Jupyter lab. Pip will decide of the right version for your package compared to the versions of the other packages present in the Docker image. Then, you can find what version pip has installed by using pip freeze, and copying/pasting the output of the command into pip_requirements.txt. You can then rebuild the Docker image, which will now contain the package you need.

Important

All functions in functionsForCalibration.py have a docstring that you can read in the file, or display in a terminal with help(<Name of the function>) ! Use it if you’re lost.

parse_PnET_ComplexTableParameterfile(file_path)
    Function used to parse a LANDIS-II parameter file that as the same
    format as the PnET species parameter file or the PnET Ecoregions parameter
    file; meaning :
        
        - A first line that indicates the type of the parameter file
          (i.e. "LandisData PnETSpeciesParameters")
        - A second line that lists all parameters for a certain entity
        - Many lines that lists the values of the parameters for several
          entities (species, ecoregions, etc.)
    
    Parameters
    ----------
    file_path : String
        Path to the LANDIS-II parameter file.
    
    Returns
    -------
    result_dict : Dictionnary
        Dictionnary containing the parameter values.
        The first entry is "LandisData":"Value"
        The second entry is another dictionnary whose key is the first word
        on the line that describe all of the parameters for the entities (i.e. PnETSpeciesParameters)
        This nested dictionnary then contains other nested dictionnaries, one
        for each entity (e.g. one per ecoregion), with the associated key of the
        name of the entity (e.g. "eco1"). These dictionnaries contain the value
        of all of the parameters for the entity,a assiciated with the name
        of the parameter as a key.
        
        Here is an example of an output dictionnary for a simple PnET ecoregion
        parameter file with one ecoregion (eco0).
        
        {'LandisData': 'EcoregionParameters',
        'EcoregionParameters':
            {'eco0':
             {'RootingDepth': '1000',
              'SoilType': 'SAND',
              'LeakageFrac': '1',
              'PrecLossFrac': '0',
              'SnowSublimFrac': '0.15',
              'ClimateFileName': 'climate.txt'}
            }
        }
            
        For example, to get parameter "SoilType" for "eco0", use :
            
            outputDict["EcoregionParameters"]["eco0"]["SoilType"].
